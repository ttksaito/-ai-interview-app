## システム概要

AIインタビューアプリケーションは、Claude AI（Sonnet 4.5）を活用した質的インタビューシステムです。転職理由テーマでは、転職を考えている人や転職経験者の本音を引き出し、5つのカテゴリ（A-E）で構造化された分析結果を提供します。

### 主要機能

- **テーマ選択**: 生きがい・人生の意味 / 転職理由
- **AIインタビュー**: 約15-20問の質的インタビュー
- **リアルタイムチャット**: WebSocketライクなストリーミング対応
- **増分分析**: メッセージごとの構造化データ抽出
- **バッチ処理**: 3メッセージ並列処理で3倍高速化
- **1対1エビデンスリンク**: 根拠とメッセージの正確な紐付け

```typescript
router.post('/message', async (req, res) => {
  const { sessionId, message } = req.body;
  const session = sessions.get(sessionId);

  // ユーザーメッセージを履歴に追加
  session.chatHistory.push({ role: 'user', content: message });

  // Claude APIから応答取得
  const { response, inputTokens, outputTokens } =
    await claudeService.getResponse(session.chatHistory, session.theme);

  // 終了コードチェック
  const endCodes = ['x7y8', '1y4x', '5j3k'];
  const isEndCode = endCodes.includes(response.trim());

  if (isEndCode) {
    session.isActive = false;
  } else {
    session.chatHistory.push({ role: 'assistant', content: response });
  }

  // トークン使用量を累積
  session.costTracker.totalInputTokens += inputTokens;
  session.costTracker.totalOutputTokens += outputTokens;

  return res.json({
    message: response,
    isActive: session.isActive,
    isEndCode,
    costTracker: session.costTracker,
  });
});
```

### Phase 3: バッチ分析

```typescript
// Frontend: InterviewScreen.tsx
const analyzeIncrementally = async () => {
  setIsAnalyzing(true);

  // トランスクリプト取得
  const { chatHistory } = await api.getTranscript(sessionId);
  const userMessageCount = chatHistory.filter(
    msg => msg.role === 'user' && msg.content !== 'インタビューを開始してください。'
  ).length;

  console.log(`Starting batch analysis for ${userMessageCount} messages`);

  // バッチ処理（3メッセージずつ）
  const BATCH_SIZE = 3;
  for (let i = 0; i < userMessageCount; i += BATCH_SIZE) {
    const batchEnd = Math.min(i + BATCH_SIZE, userMessageCount);
    console.log(`Analyzing batch: messages ${i + 1}-${batchEnd}`);

    // 並列処理
    const batchPromises = [];
    for (let j = i; j < batchEnd; j++) {
      batchPromises.push(api.analyzeMessage(sessionId, j));
    }
    await Promise.all(batchPromises);

    // 進捗更新
    setAnalyzingProgress({ current: batchEnd, total: userMessageCount });
  }

  // 分析結果の集約
  await api.finalizeAnalysis(sessionId);
};

// API Requests (並列実行)
POST /api/interview/analyze-message
Body: { sessionId: 'session_...', messageIndex: 0 }
POST /api/interview/analyze-message
Body: { sessionId: 'session_...', messageIndex: 1 }
POST /api/interview/analyze-message
```

```typescript
// analysisService.ts
async analyzeMessage(
  conversationContext: Message[],
  latestMessage: Message,
  messageIndex: number,
  theme: InterviewTheme = 'life-meaning',
): Promise<MessageAnalysis> {
  const itemTexts = getItemTexts(theme);

  // 直近5メッセージを文脈として使用
  const contextMessages = conversationContext.slice(-5);
  const conversationText = contextMessages
    .map(msg => {
      const role = msg.role === 'assistant' ? 'AIインタビュアー' : '回答者';
      return `${role}: ${msg.content}`;
    })
    .join('\n\n');

  // 分析用プロンプト作成
  const analysisPrompt = `
以下の会話文脈において、最新の回答者の発言を分析してください。

## 会話の文脈:
${conversationText}

## 分析対象（回答者の最新発言）:
${latestMessage.content}

## 評価項目:
${Object.entries(itemTexts).map(([id, text]) => `${id}: ${text}`).join('\n')}

## 出力形式:
各項目について以下のJSON形式で評価してください：
{
  "カテゴリID_項目番号": {
    "evaluation": 1 or -1 or 0,
    "evidence": "発言からの引用（evaluation が 0 の場合は「なし」）"
  }
}

evaluation の意味:
- 1: ポジティブ（満足している、できている）
- -1: ネガティブ（不満、できていない）
- 0: 言及なし

evidence は、回答者の発言から該当箇所を「」で囲んで引用してください。
`;

  // Claude APIで分析実行
  const response = await this.client.messages.create({
    model: 'claude-sonnet-4-5',
    max_tokens: 4096,
    messages: [{ role: 'user', content: analysisPrompt }],
  });

  // JSON抽出とパース
  const responseText = response.content[0].text;
  const jsonMatch = responseText.match(/```json\s*([\s\S]*?)\s*```/);
  const jsonText = jsonMatch ? jsonMatch[1] : responseText;
  const parsedData = JSON.parse(jsonText);

  // MessageAnalysis形式に変換
  const categories: { [key: string]: any[] } = {};
  for (const [itemId, data] of Object.entries(parsedData)) {
    const categoryId = itemId.charAt(0); // 'A', 'B', 'C', 'D', 'E'

    if (!categories[categoryId]) {
      categories[categoryId] = [];
    }

    categories[categoryId].push({
      itemId,
      evaluation: data.evaluation,
      evidence: data.evidence === 'なし' ? '' : data.evidence,
    });
  }

  return {
    messageIndex,
    categories,
  };
}
```

**aggregateMessageAnalyses() の処理:**

```typescript
// analysisService.ts
aggregateMessageAnalyses(
  messageAnalyses: MessageAnalysis[],
  theme: InterviewTheme,
  transcript: string,
): AnalysisResult {
  const itemTexts = getItemTexts(theme);

  // 全50項目を初期化（evaluation: 0）
  const allItems: { [key: string]: AnalysisItem } = {};
  ['A', 'B', 'C', 'D', 'E'].forEach(cat => {
    for (let i = 1; i <= 10; i++) {
      const itemId = `${cat}${i}`;
      allItems[itemId] = {
        id: itemId,
        item: itemTexts[itemId],
        mentions: [],
        evaluation: 0,
        evidence: '言及なし',
      };
    }
  });

  // 各メッセージ分析を統合
  messageAnalyses.forEach(msgAnalysis => {
    Object.entries(msgAnalysis.categories).forEach(([categoryId, items]) => {
      items.forEach(item => {
        if (item.evaluation !== 0) {
          // 評価を更新（最新の評価を優先）
          allItems[item.itemId].evaluation = item.evaluation;

          // エビデンスを追加
          if (item.evidence && item.evidence !== 'なし') {
            const formattedEvidence = item.evidence.startsWith('「')
              ? `【${item.evidence}と発言】`
              : `【「${item.evidence}」と発言】`;

            // mentions配列に追加（1対1リンク用）
            allItems[item.itemId].mentions.push({
              messageIndex: msgAnalysis.messageIndex,
              evaluation: item.evaluation,
              evidence: formattedEvidence,
              messageContent: '',
            });

            // 統合エビデンス文字列（カンマ区切り）
            if (allItems[item.itemId].evidence === '言及なし') {
              allItems[item.itemId].evidence = formattedEvidence;
            } else {
              allItems[item.itemId].evidence += ', ' + formattedEvidence;
            }
          }
        }
      });
    });
  });

  // カテゴリごとにグループ化
  const categorizedItems = { A: [], B: [], C: [], D: [], E: [] };
  Object.values(allItems).forEach(item => {
    const categoryId = item.id.charAt(0);
    categorizedItems[categoryId].push(item);
  });

  // カテゴリオブジェクト作成
  const categoryNames = getCategoryNames(theme);
  return {
    categories: {
      A: this.createCategory('A', categoryNames.A, categorizedItems.A),
      B: this.createCategory('B', categoryNames.B, categorizedItems.B),
      C: this.createCategory('C', categoryNames.C, categorizedItems.C),
      D: this.createCategory('D', categoryNames.D, categorizedItems.D),
      E: this.createCategory('E', categoryNames.E, categorizedItems.E),
    },
    transcript,
  };
}

private createCategory(
  id: string,
  name: string,
  items: AnalysisItem[],
): AnalysisCategory {
  const positiveCount = items.filter(item => item.evaluation === 1).length;
  const negativeCount = items.filter(item => item.evaluation === -1).length;

  return { name, items, positiveCount, negativeCount };
}
```

## バッチ処理の仕組み

### なぜバッチ処理が必要か？

**逐次処理の問題:**
```
メッセージ1 → 分析（10秒） → 完了
メッセージ2 → 分析（10秒） → 完了
メッセージ3 → 分析（10秒） → 完了
...
15メッセージ → 合計150秒（2分30秒）
```

**バッチ処理の利点:**
```
バッチ1: メッセージ1,2,3 → 並列分析（10秒） → 完了
バッチ2: メッセージ4,5,6 → 並列分析（10秒） → 完了
バッチ3: メッセージ7,8,9 → 並列分析（10秒） → 完了
...
15メッセージ → 合計50秒（約3倍高速化）
```

### バッチサイズの決定

**考慮要素:**
1. **Vercel関数タイムアウト**: 60秒
2. **Claude APIレート制限**: 並列リクエスト数
3. **1メッセージあたりの分析時間**: 約8-10秒

**バッチサイズ3を選択した理由:**
- 3メッセージ × 10秒 = 30秒 < 60秒（タイムアウト）
- レート制限に余裕
- メモリ使用量が適切

### バッチ処理のコード詳細

```typescript
const BATCH_SIZE = 3;

for (let i = 0; i < userMessageCount; i += BATCH_SIZE) {
  const batchEnd = Math.min(i + BATCH_SIZE, userMessageCount);

  // このバッチで処理するメッセージのPromiseを作成
  const batchPromises = [];
  for (let j = i; j < batchEnd; j++) {
    batchPromises.push(api.analyzeMessage(sessionId, j));
  }

  // 並列実行して全て完了を待つ
  await Promise.all(batchPromises);

  // 進捗更新
  setAnalyzingProgress({ current: batchEnd, total: userMessageCount });
}
```

### タイムアウト管理

**Vercel設定 (vercel.json):**
```json
{
  "functions": {
    "api/index.ts": {
      "memory": 1024,
      "maxDuration": 60
    }
  }
}
```

**推奨メッセージ数:**
- 6メッセージ（2バッチ）: ~20秒 ✅ 安全
- 9メッセージ（3バッチ）: ~30秒 ✅ 安全
- 12メッセージ（4バッチ）: ~40秒 ✅ まだ余裕
- 15メッセージ（5バッチ）: ~50秒 ✅ ギリギリOK
- 18メッセージ（6バッチ）: ~60秒 ⚠️ タイムアウトリスク

## パフォーマンス最適化

### 1. バッチ並列処理

**効果:** 分析時間を約1/3に短縮

**実装:**
```typescript

  api.analyzeMessage(sessionId, 0),
```

### 2. 増分分析

**メリット:**
- メッセージごとに小さな分析タスク
- タイムアウトリスクの低減
- 進捗の可視化

**vs 一括分析:**
- 一括: 全メッセージを1回のClaude API呼び出し → タイムアウトリスク高
- 増分: メッセージごとに小さく分析 → 安定性高

```typescript
const contextMessages = conversationContext.slice(-5);
```

**メリット:**

- 処理速度向上
- 文脈の関連性維持

```typescript
const sessions = new Map<string, InterviewSession>();
```

**メリット:**

- 複数インスタンス間で共有不可
